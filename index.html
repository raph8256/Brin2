# 🌿 Brin — La question du jour, rien que pour vous deux Application iOS native (SwiftUI) qui envoie chaque matin à 6h30 une question générée par IA à vous et votre partenaire. Vous répondez dans un fil de chat partagé, intime et privé. --- ## ✨ Fonctionnalités - 📬 Notification à 6h30 avec la question du jour - 🤖 Questions générées par Claude (Anthropic) sur des thèmes variés - 💬 Chat partagé autour de chaque question, comme une conversation privée - 📅 Historique de toutes vos questions et réponses - 🔐 Connexion par email - 👫 Liaison des comptes via un code à 6 chiffres --- ## 🛠 Stack technique | Couche | Technologie | |--------|-------------| | UI | SwiftUI (iOS 17+) | | Backend / DB | Firebase Firestore | | Auth | Firebase Authentication | | Notifications | APNs via Firebase Cloud Messaging | | IA | API Anthropic (Claude Sonnet) | | Tâches planifiées | Firebase Cloud Functions v2 | --- ## 📋 Prérequis - Xcode 15 ou plus - iOS 17 minimum - Un compte [Firebase](https://console.firebase.google.com) (gratuit) - Une clé API [Anthropic](https://console.anthropic.com) (gratuit au début) - Un compte Apple Developer (99€/an — pour TestFlight + notifications push) --- ## 🚀 Installation étape par étape ### 1. Cloner le repo ```bash git clone https://github.com/TON_USERNAME/Brin.git cd Brin ``` ### 2. Créer le projet Firebase 1. Aller sur [console.firebase.google.com](https://console.firebase.google.com) 2. Créer un nouveau projet → donner un nom (ex: `brin-app`) 3. Ajouter une app iOS : - Bundle ID : `com.tonnom.Brin` - Télécharger `GoogleService-Info.plist` - Le glisser dans le dossier `Brin/Brin/` dans Xcode 4. Activer **Authentication** → Email/Password 5. Activer **Cloud Firestore** → mode Production 6. Activer **Cloud Messaging** ### 3. Ajouter Firebase via Swift Package Manager Dans Xcode : - `File` → `Add Package Dependencies` - URL : `https://github.com/firebase/firebase-ios-sdk` - Cocher : `FirebaseAuth`, `FirebaseFirestore`, `FirebaseMessaging` ### 4. Déployer les Cloud Functions ```bash # Installer Firebase CLI si pas encore fait npm install -g firebase-tools firebase login cd functions npm install # Ajouter ta clé Anthropic (jamais exposée côté client) firebase functions:secrets:set ANTHROPIC_API_KEY # Coller ta clé quand demandé # Déployer cd .. firebase deploy --only functions ``` ### 5. Configurer les notifications push (APNs) 1. Sur [developer.apple.com](https://developer.apple.com) → Certificates → Keys → créer une **APNs Key** (.p8) 2. Dans Firebase Console → Project Settings → Cloud Messaging → Apple app config → uploader la clé .p8 ### 6. Lancer l'appli - Ouvrir `Brin.xcodeproj` dans Xcode - Sélectionner ton iPhone comme cible - `Cmd + R` pour lancer --- ## 📁 Structure du projet ``` Brin/ ├── Brin/ │ ├── App/ │ │ ├── BrinApp.swift # Entry point │ │ └── AppDelegate.swift # Firebase + APNs │ ├── Models/ │ │ └── Models.swift # User, Couple, Question, Message │ ├── Views/ │ │ ├── Auth/ │ │ │ ├── LoginView.swift │ │ │ └── PairingView.swift │ │ ├── Today/ │ │ │ └── TodayView.swift # Question + Chat │ │ ├── History/ │ │ │ └── HistoryView.swift │ │ └── MainTabView.swift │ ├── ViewModels/ │ │ ├── AuthViewModel.swift │ │ └── TodayViewModel.swift │ └── Services/ │ ├── FirestoreService.swift │ └── NotificationService.swift ├── functions/ │ ├── index.js # Cron + génération IA │ └── package.json ├── firestore.rules ├── firestore.indexes.json ├── firebase.json ├── .gitignore └── README.md ``` --- ## 🧠 Comment ça marche Chaque matin à **6h29** (heure de Paris), Firebase Cloud Scheduler déclenche une fonction qui : 1. Choisit un thème aléatoire (couple, futur, enfants, société, intimité, rêves, philo) 2. Génère une question via l'API Claude 3. Sauvegarde dans Firestore sous `/questions/YYYY-MM-DD` 4. Envoie une notification push aux deux iPhones du couple --- ## 📲 Installer sur vos deux iPhones (TestFlight) 1. Dans Xcode → `Product` → `Archive` 2. Uploader sur App Store Connect 3. Activer TestFlight → inviter vos deux emails 4. Installer l'appli depuis l'app TestFlight sur chaque iPhone --- ## 🔒 Règles de sécurité Chaque couple n'accède qu'à ses propres données. Voir `firestore.rules`. --- ## 📄 Licence MIT — Fait avec 🌿 pour vous deux.
